# Non-Functional Requirements Register — carbonyl-agent

**Status**: Baselined v1.0, 2026-04-09
**Scope**: Python automation SDK. NFRs apply to the SDK itself; performance of the underlying Carbonyl runtime is out of scope except where the SDK materially affects it (PTY throughput, drain loops).

Priority legend: **MUST** (0.1 blocker), **SHOULD** (0.1 desired), **COULD** (post-0.1).

| NFR-ID  | Category        | Requirement                                                                                                                                                                         | Measurement                                                                                           | Priority | Verification                                                            |
|---------|-----------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------|----------|-------------------------------------------------------------------------|
| NFR-01  | Performance     | `CarbonylBrowser.drain(s)` must sustain PTY reads at ≥1 MiB/s on a modern laptop without dropping bytes from the pyte stream (`browser.py:drain`, 8 KiB chunks, 100 ms poll).       | Benchmark against a synthetic PTY producer streaming 10 MiB; assert full reconstruction of input.    | SHOULD   | Integration benchmark in `tests/`                                       |
| NFR-02  | Performance     | Daemon RPC round-trip latency (`DaemonClient._rpc` excluding drain) must be <20 ms p95 on localhost Unix socket for a nontrivial command (`page_text`).                             | Measure 1k RPCs, report p50/p95/p99.                                                                  | SHOULD   | Integration benchmark                                                   |
| NFR-03  | Reliability     | Daemon must survive client disconnect mid-request without leaking file descriptors or threads (`daemon.py:_BrowserHandler.handle`).                                                 | Stress test: 1k connect/disconnect cycles, assert FD count and thread count return to baseline.       | SHOULD   | Integration test                                                        |
| NFR-04  | Reliability     | After a crashed Chromium process, the next `CarbonylBrowser(session=...).open()` must succeed without manual intervention by cleaning the stale `SingletonLock` (`session.py`).     | Kill -9 a running browser, reopen same session, assert success.                                      | MUST     | Integration test                                                        |
| NFR-05  | Compatibility   | SDK must import and run on Python 3.11 and 3.12 (CPython).                                                                                                                          | CI matrix runs `pytest tests/` on both versions.                                                      | MUST     | CI                                                                      |
| NFR-06  | Compatibility   | Supported platforms: `x86_64-unknown-linux-gnu`, `aarch64-unknown-linux-gnu`, `x86_64-apple-darwin`, `aarch64-apple-darwin` (matches `install.py:_platform_triple`).                 | Install smoke test (`carbonyl-agent install` + `status`) passes on each target.                      | MUST     | Release checklist                                                       |
| NFR-07  | Security        | Runtime tarball download must be verified against a SHA256 manifest published in the same Gitea release before extraction (`install.py:cmd_install`).                               | Tamper test: corrupt downloaded tarball → install fails with checksum mismatch.                       | SHOULD   | Unit test + release process                                             |
| NFR-08  | Security        | Daemon Unix socket must be created with mode 0600 (owner-only RW) and live inside a user-writable session directory — no world access.                                             | Stat the socket after `start_daemon`; assert mode `0o600`.                                             | MUST     | Unit test                                                               |
| NFR-09  | Security        | SDK must not execute code downloaded at install time beyond extracting the tarball and chmod-ing the binary; no post-install scripts.                                               | Code review + `install.py` audit.                                                                     | MUST     | Code review checklist                                                   |
| NFR-10  | Maintainability | Test coverage for `browser.py`, `session.py`, `daemon.py`, `install.py`, `screen_inspector.py` ≥80% line coverage.                                                                  | `pytest --cov=src/carbonyl_agent --cov-fail-under=80`.                                                | SHOULD   | CI gate                                                                 |
| NFR-11  | Maintainability | All public API functions and classes carry type hints and docstrings; `__all__` defined in `__init__.py`.                                                                           | Static: `mypy --strict src/carbonyl_agent` + `ruff D` doc rules.                                      | SHOULD   | CI lint                                                                 |
| NFR-12  | Portability     | Binary discovery must follow documented order: `CARBONYL_BIN` → install dir → `$PATH` → Docker fallback; each step must be independently testable via env manipulation.            | Unit tests mock each layer and assert resolution order.                                               | MUST     | Unit test                                                               |
| NFR-13  | Portability     | Docker fallback must work with no local binary and no `CARBONYL_BIN` set, using the `fathyb/carbonyl` image.                                                                        | Manual / CI job with `docker` available, binary absent.                                              | COULD    | Manual release test                                                     |
| NFR-14  | Observability   | SDK must emit structured log lines via `browser.log()` prefixed `[carbonyl]` to stderr for all lifecycle events (open, reconnect, daemon start/stop, fallback).                     | Grep for required event lines in captured stderr during integration tests.                            | SHOULD   | Integration test                                                        |
| NFR-15  | Observability   | A debug / verbose mode (env flag e.g. `CARBONYL_DEBUG=1`) should surface per-RPC timing and PTY byte counts without code changes.                                                   | Manual: set flag, run a scripted scenario, inspect stderr.                                            | COULD    | Manual                                                                  |
| NFR-16  | Usability       | Public API (`CarbonylBrowser`, `SessionManager`, `ScreenInspector`, `DaemonClient`) must be importable from the top-level `carbonyl_agent` package and documented in the README.   | Doc review; `from carbonyl_agent import …` smoke test.                                                | MUST     | Smoke test                                                               |
| NFR-17  | Usability       | CLI (`carbonyl-agent install|status`) must produce actionable error messages on failure (bad tag, network error, missing binary) and exit with non-zero status.                    | Negative tests invoking CLI with bad args / unreachable host.                                         | SHOULD   | Integration test                                                        |
| NFR-18  | Licensing       | SDK is distributed under MIT; all runtime dependencies (`pexpect`, `pyte`) must have MIT- or BSD-compatible licenses. Third-party runtime (Carbonyl) license noted separately.      | License audit via `pip-licenses` at release time.                                                     | MUST     | Release checklist                                                       |
